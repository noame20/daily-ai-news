1 VoltAgent/awesome-design-md

收集了 69+ 个来自真实网站的设计系统文档（DESIGN.md），源自 Google Stitch 提出的纯文本设计规范格式。每个文件涵盖：色彩体系、字体规范、组件样式、布局原则、深度层级、设计禁区、响应式策略等 9 大维度。

核心价值：AI 编码 agent 读取 DESIGN.md 后可直接生成风格一致的 UI，无需 Figma 导出或 JSON 配置。覆盖 AI/LLM 平台（Claude、OpenAI、Cohere 等）、开发工具（Cursor、Vercel、Raycast）、金融（Stripe、Mastercard、Wise）、汽车（Tesla、BMW、法拉利）等垂直领域。

亮点是 AI 工作流中"设计语言传递"这一环终于有了标准解法——设计师出 DESIGN.md，AI agent 直接消费，无缝衔接。

2 openai/openai-agents-python

OpenAI 开源的多 Agent 工作流框架(provider-agnostic，支持 100+ LLM)。核心概念：Agent（带工具/guardrails/handoffs）、Sandbox Agent（带容器可长时间运行）、Agents as Tools/Handoffs、Tracing、Realtime Agent（语音）等。

v0.14.0 新增 Sandbox Agent——在配置好的容器环境中真实执行代码（文件系统操作、命令运行、补丁应用），跨长周期任务保持工作区状态。结合 MCP 协议支持，定位是轻量但专业的 Agent 编排基础设施。

3 NanmiCoder/cc-haha

基于 Claude Code 泄露源码的本地可运行版本，新增：跨平台桌面端（Tauri 2 + React）、远程控制（支持 Telegram / 飞书 / Discord）、记忆系统、多 Agent 系统、Skills 系统、Computer Use 桌面控制等。

架构基于 Bun + TypeScript + Ink（TUI）+ Anthropic SDK，支持接入任意 Anthropic 兼容 API（MiniMax、OpenRouter 等）。本地化后补全了官方未公开的 Computer Use 部分（原本只在上线版本中）。适合需要本地运行 Agent 且不愿受限于官方云服务的用户。

4 webadderall/Recordly

开源屏幕录制 + 编辑工具，目标是替代 Screen Studio。技术栈：Electron + PixiJS + 原生平台捕获 API（macOS ScreenCaptureKit / Windows WGC）。

核心功能：自动 zoom 建议、光标美化（模糊/弹跳/摇摆）、时间线剪辑、摄像头叠加、帧边框装饰（壁纸/渐变/模糊/阴影）、导出 MP4/GIF。社区扩展系统已上线，可接入光标音效、设备框架、浏览器 mockup 等插件。

5 vercel-labs/wterm

Vercel 出品的 Web 端终端模拟器，核心用 Zig 编写并编译为 WASM（约 12KB 二进制），解析 VT100/VT220/xterm escape sequences，渲染到 DOM。DOM 渲染方式天然支持原生文本选择、复制粘贴、浏览器查找和无障碍访问。

内置 Dirty-row tracking（仅重绘变动行）、24-bit 颜色、Alternate screen buffer（vim/less/htop 完美支持）、滚动历史、WebSocket 传输（连接后端 PTY）。包体系：@wterm/core（无头 WASM 桥）、@wterm/dom（纯 JS 渲染器）、@wterm/react（React 组件+hooks）、@wterm/just-bash（内置 Bash shell）。

6 elder-plinius/CL4R1T4S

收集并公开各大 AI 厂商（OpenAI、Google、Anthropic、xAI、Perplexity、Cursor、Windsurf、Devin、Manus、Replit 等）的系统提示词（System Prompt），项目名即 "Claritas"（拉丁语"透明"）。核心主张："要信任输出，必须理解输入"——AI 厂商通过隐藏的 prompt 脚手架塑造模型行为，用户若不了解系统提示词就无法判断 AI 输出是否被干预。

以 PR 形式接受各模型的 system prompt 提交，需提供模型名称/版本、提取日期等信息。属于 AI 透明度与可观测性基础设施类项目。
