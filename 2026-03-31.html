1 OpenBB-finance/OpenBB

开源金融数据平台，为分析师、量化交易员和 AI Agents 提供统一的数据接入层。核心理念是"连接一次，处处消费"——通过统一接口整合多个金融数据源，支持 Python SDK、CLI、REST API、Excel 和 MCP 服务器等多种消费方式。内置 FastAPI 驱动的高性能 API 服务器，可直接对接 AI Copilot 应用。企业版 OpenBB Workspace 提供可视化仪表板。

2 chenglou/pretext

纯 JavaScript/TypeScript 的多行文本测量与布局库，无需触碰 DOM 即可精确测量文本高度，完全避免浏览器布局重排（这是最昂贵的 DOM 操作之一）。核心原理是两阶段架构：`prepare()` 用 Canvas 测量每个词/字素宽度并缓存，`layout()` 阶段仅做纯算术运算（约 0.09ms 处理 500 个文本块）。内置 Intl.Segmenter 分词，支持 CJK、泰语、阿拉伯语混合 bidi 等所有语言。广泛用于虚拟列表、Masonry 布局、AI 布局验证（防止按钮标签溢出）和 CLS 优化。
⭐Stars 21732 🍴Forks 872 🔀PR 24 🔖Issue 24

3 EveryInc/compound-engineering-plugin

**复利工程（Compound Engineering）**理念的 Claude Code 插件，核心理念：每个工程单元的工作都应该使后续工作更容易。工作流：Brainstorm → Plan → Work → Review → Compound → Repeat。内置 35+ 专业化 Agents 和 40+ Skills，通过 6 大架构模式（Pipeline、Fan-out/Fan-in、Expert Pool、Producer-Reviewer、Supervisor、Hierarchical Delegation）组织多智能体协作。支持 Claude Code、Cursor、Codex、GitHub Copilot、Windsurf 等多平台，配备 context7 MCP 服务器支持 100+ 框架文档查询。实验数据：15 项软件工程任务中质量分数提升 +60%，输出方差降低 -32%。
⭐Stars 12066 🍴Forks 932 🔀PR 20 🔖Issue 34

4 google-research/timesfm

Google Research 开发的时间序列基础模型（ICML 2024 论文），Decoder-only 架构，支持零样本时间序列预测。最新版本 TimesFM 2.5：200M 参数，16k 上下文长度，Patch-based 输入（32 时间点分块），旋转位置编码（RoPE）。支持点预测和分位数预测（0.1~0.9），内置 RevIN 归一化提升泛化能力，Scale 不变性保证。已集成至 Google BigQuery，原生支持 PyTorch/Flax 双框架，可部署至 CPU/GPU/TPU/Apple Silicon。
⭐Stars 10977 🍴Forks 929 🔀PR 43 🔖Issue 135

5 vas3k/TaxHacker

自托管 AI 会计应用，通过 AI 自动识别发票、收据等财务文档（提取日期、金额、商品明细、税率等），支持 170+ 法定货币和 14 种加密货币（基于历史汇率自动换算）。支持无限自定义字段、类别和项目，以及完全自定义的 AI 提示词。技术栈：Next.js 15 + Prisma + PostgreSQL 17 + LangChain，支持 OpenAI GPT、Google Gemini、Mistral。Docker 一键部署，数据完全自主可控。

趋势分析

历史关联检索结果

基于 qmd 历史日报检索，发现以下关联趋势：

  • Oracle AI Database 26ai（2026-03-28）：推出统一内存核心，支持向量/JSON/图/关系型等混合数据访问，配合 Private Agent Factory（无代码 AI 代理构建平台），解决 AI 代理需要拼接多个数据库的痛点
  • Nvidia NemoClaw（2026-03-18）：企业级 autonomous agents 安全运行工具包，在隔离虚拟环境中运行 AI 代理
  • Foundation Model 趋势：近期多篇日报涉及音视频生成基础模型（daVinci-MagiHuman）、时间序列基础模型（TimesFM）等，Foundation Model 正在向垂直领域加速渗透

技术演进分析

  1. AI Agent 基础设施层正在快速补全：从 Oracle 的统一内存核心到 Nvidia 的 NemoClaw，再到今天的 Harness（Agent 团队设计工具），行业正在从"单点能力"向"系统性基础设施"演进。今日 OpenBB 的 MCP 服务器支持也是同一脉络——数据平台在 AI 时代的必要升级

  2. 零样本预测进入实用化阶段：TimesFM 2.5 从 512 上下文升级到 16k，并支持连续分位数预测头，零样本时间序列预测正从研究走向生产。Oracle 26ai 的 Autonomous AI Vector Database 也印证了这一方向

  3. 浏览器文本处理性能优化受到关注：沉寂已久的 Pretext 项目（21k stars）今日冲上 TrendShift，说明在 AI驱动的动态 UI 场景下，DOM 性能问题重新成为焦点。虚拟列表、Masonry 布局、CLS 优化等场景对高精度文本测量的需求正在增长

风向观察

  • Agent 编排工具竞争加剧：Harness（+60% 质量提升）和 Compound Engineering Plugin（复利工程理念）代表了两种不同的 Agent 团队协作范式，预计接下来会出现更多类似工具和基准测试对比

行动建议

  1. 关注 MCP 生态：OpenBB 和 fli 都通过 MCP 将自身能力接入 AI Agent，MCP 已成为 AI 时代数据/服务接入标准，nagi 应重点关注 MCP Server 的开源生态
  2. 跟进 TimesFM 生产落地：TimesFM 已集成 BigQuery，预计接下来会有更多数据库/云平台原生支持，零样本预测的工程化路径值得关注
  3. 关注文本布局性能：在 AI 驱动的动态 UI 趋势下，Pretext 这类库的思路值得借鉴，nagi 可关注其在实际产品中的采纳情况
