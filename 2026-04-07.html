1 NousResearch/hermes-agent

自我进化的 AI Agent，由 Nous Research打造。核心特点是内置学习循环——能从经验中创建技能（skills）、使用中自我改进、自主记忆整理，以及跨会话持续追踪用户画像。支持 6 种终端后端（本地、Docker、SSH、Daytona、Singularity、Modal），可在 $5 VPS 或 GPU 集群上运行。调度层面内置 cron，支持 Telegram/Discord/Slack/WhatsApp 等多平台消息网关。兼容 agentskills.io 开放标准，可连接任意 MCP 服务器。

2 JackChen-me/open-multi-agent

TypeScript 多智能体框架，只需一次 `runTeam()` 调用即可完成从目标到结果的完整流程。框架自动分解任务、解析依赖关系、并行执行独立任务，最终合成输出。运行时仅 3 个依赖（`@anthropic-ai/sdk`、`openai`、`zod`），33 个源文件，可在任何 Node.js 环境中部署。支持 Claude/GPT/Gemma 4/本地模型（Ollama、vLLM、LM Studio），内置任务重试、循环检测、人工审批节点、可观测性追踪等企业级特性。被 Latent Space AI News 提及。
⭐Stars 5179 🍴Forks 2127 🔀PR 3 🔖Issue 17

3 JuliusBrussee/caveman

Claude Code / Codex 技能插件，通过"穴居人说话风格"将输出 token 削减约 65%，同时保持技术准确性。内置 3 档强度（Lite / Full / Ultra），用户可自主选择压缩幅度。附赠 `caveman compress` 工具可将记忆文件输入 token 削减约 45%。配套论文引用 2026 年 3 月 arXiv 论文 "Brevity Constraints Reverse Performance Hierarchies in Language Models"，证明约束大型模型简洁回复可将某些基准准确率提升 26 个百分点。

趋势分析

核心关键词：多 Agent 记忆持久化、TypeScript Agent 框架、Token 压缩量化、eGPU 本地推理、活体神经机器人

技术连贯性：

  • hermes-agent 的内置学习循环 + 技能系统，与 2026-03-27 出现的 `oh-my-claudecode`（多 Agent 编排框架，Team 模式）形成同一技术路线的两端：oh-my-claudecode 侧重团队协作流程编排，hermes-agent 侧重个体 Agent 的长期记忆与技能积累。两者共同指向"有状态、可累积经验的多 Agent 系统"正在成为开源主流。
  • caveman 的 Token 压缩思路，与近期 Claude Code 系统提示泄露事件形成侧面印证：当用户开始深度定制 Agent 行为（如压缩输出），实际上也在自发地精简 System Prompt / Context Window，说明 Token 效率已从"模型侧优化"渗透到"应用侧实践"。
  • open-multi-agent 作为 TypeScript 原生框架出现，与传统 Python Agent 生态形成语言栈互补，降低了前端/全栈工程师接入 Agent 开发的门槛。

风向信号：

  • 苹果批准 eGPU 驱动是一个硬件侧信号：Mac OS 正式打开外部 GPU 用于 AI 推理的通路，意味着消费级硬件+本地大模型推理的可行性提升，对 llama.cpp/Ollama 类本地推理工具有积极影响。
  • 活体神经机器人（Neurobots）是AI × 生物交叉的早期探索，虽距实际应用遥远，但代表了一种"非硅基 AI 载体"的新思路。

预测与行动建议（未来两周）：

  1. 多 Agent 协作框架热度将持续，关注点从"能否工作"转向"如何管理长期记忆/跨会话状态"。hermes-agent 的技能自改进机制值得重点研究，其与 agentskills.io 开放标准的结合可能是下一个生态爆发点。
  2. Token 效率工具链（压缩、摘要、上下文管理）将成为 Agent 开发的标配，caveman 类方案可能催生更多专注"输出风格控制"的中间件。
  3. eGPU + 本地推理组合值得关注，若苹果生态走通，未来可能看到"MacBook + eGPU + 70B 模型"作为低成本开发方案的普及。
  4. TypeScript Agent 框架（open-multi-agent）填补了 JS/TS 生态空白，对已在使用 TypeScript 的团队（特别是 Next.js、Express 等场景）是低迁移成本的接入点。
